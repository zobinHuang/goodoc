# 07 · 国际化（i18n）

goodoc 用**locale 前缀路由**实现 i18n（`/en/…`、`/zh/…`），默认 English。这样无需
middleware/运行时重定向，保持纯静态导出。

## 组成

| 关注点 | 位置 |
| --- | --- |
| locale 列表 / 默认 / 工具 | `lib/i18n.ts`（`locales`、`defaultLocale`、`localePath` 等） |
| UI 文案（框架 chrome） | `lib/dictionaries.ts`（`Dictionary` 类型 + en/zh） |
| 项目文案（hero/features/nav…） | `lib/site-config.ts` 的 `locales[lang]` |
| 内容 | `content/<locale>/{docs,blog}/*.md` |
| 日期格式 | `lib/format.ts`（按 locale） |

## 路由

所有页面在 `app/[lang]/` 下。`app/[lang]/layout.tsx` 用 `generateStaticParams`
枚举 locale；子 `[...slug]` 路由的 `generateStaticParams` 接收父级的 `{ lang }`，
为该 locale 返回 slug 列表：

```tsx
export function generateStaticParams({ params }: { params: { lang: string } }) {
  const lang = resolveLocale(params.lang);
  return getAllDocs(lang).map((doc) => ({ slug: doc.slug }));
}
```

- 根 `/`（`app/page.tsx`）渲染 `RootRedirect`，客户端跳转到 `/<defaultLocale>/`，
  并提供 no-JS 的语言选择链接。
- `<html lang>` 由根布局给默认值，`components/html-lang.tsx` 在客户端按 locale 校正。

## 内容回退

`lib/content.ts` 的 `loadCollection(lang, …)` 先加载默认 locale，再用目标 locale
覆盖同名 slug。因此某篇文档**只在默认 locale 存在**时，其它 locale 会回退到默认
版本——保证每个 locale 的路由集合完整。

## 内容里的链接与图片

`renderMarkdown(md, { lang })` 的链接重写插件按是否带文件扩展名区分：

- **路由**（无扩展名，如 `/docs/humanize/x/`）→ 加 `basePath` + locale 前缀；
- **资源**（有扩展名，如 `/logo.svg`）→ 只加 `basePath`，不本地化。

所以内容作者写 **locale-relative** 链接（`/docs/...`），渲染时自动补上当前 locale。

## 加一个 locale

1. `lib/i18n.ts`：在 `locales`、`localeNames`、`localeHtmlLang` 增加该 locale。
2. `lib/dictionaries.ts`：补一份 `Dictionary`。
3. `lib/site-config.ts`：在 `locales` 补一份 `LocaleContent`。
4. （可选）`content/<locale>/` 放翻译；缺失的文档自动回退默认 locale。

类型系统会强制你补全 1–3，缺哪块编译期就会报错。
