# 国际化

goodoc 开箱即支持多语言。URL 带语言前缀（`/en/…`、`/zh/…`），英语为默认语言，
页眉中有一个语言切换器。因为一切都是静态生成的，所以不涉及任何服务器或重定向。

## 内容如何组织

内容存放在 `content/<locale>/` 之下：

```text
content/
├── en/            # 默认语言
│   ├── docs/
│   └── blog/
└── zh/
    ├── docs/
    └── blog/
```

仅存在于默认语言的文档会在其他语言下**回退**到默认语言，因此你可以增量翻译 ——
未翻译的页面在 `/zh/…` 下仍会渲染（以英语显示）。

## 内容中的链接

链接请用**语言相对**的方式书写：

```markdown
See [Writing Content](/docs/humanize/writing-content/).
```

渲染时会自动加上当前语言前缀和部署的 base path，因此链接会解析为
`/en/docs/humanize/writing-content/`（或 `/zh/…`）。
带文件扩展名的资源（例如 `/logo.svg`）只会加上 base path —— 它们永远不会被本地化。

## 添加一种语言

三处小改动，你就拥有了一个新语言：

1. **`lib/i18n.ts`** —— 把语言代码加入 `locales`，并补上对应的 `localeNames` 和
   `localeHtmlLang` 条目。
2. **`lib/dictionaries.ts`** —— 添加一个 `Dictionary` 块，包含界面字符串（侧边栏
   标签、“On this page”、视图切换等）。
3. **`lib/site-config.ts`** —— 在 `locales` 下添加一个 `LocaleContent` 块（hero、
   features、quickstart、nav、页脚文案）。

然后把翻译放进 `content/<new-locale>/`。任何你跳过的部分都会回退到默认语言。在三步
都完成之前，TypeScript 都会报错提示，所以你不会漏掉任何一块。

## 各项内容所在位置

| 关注点 | 位置 |
| --- | --- |
| 语言列表 / 默认语言 / 辅助函数 | `lib/i18n.ts` |
| 界面框架字符串 | `lib/dictionaries.ts` |
| 项目文案（hero、features、nav…） | `lib/site-config.ts`（`locales`） |
| 内容 | `content/<locale>/{docs,blog}/` |
