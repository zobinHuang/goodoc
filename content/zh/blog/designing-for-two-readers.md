# 为两种读者而设计

文档的读者正在悄悄改变。过去，读你 README 的几乎都是人；现在，越来越多是
agent——它们替人检索、总结、回答关于你项目的问题。

可大多数站点只为人渲染。漂亮的排版背后，是层层嵌套的 `div`、脚本与样式，机器要从中
准确还原内容并不容易。

## 一份内容，两种呈现

goodoc 的做法是：**同一份 Markdown，渲染两次**。

- **humanize** 视图为人优化：文艺排版、代码高亮、目录与侧边栏。
- **agent** 视图为机器优化：去掉一切外壳，只留语义化 HTML，所有图片与文字内联。

两者来自同一条渲染管线，所以永远一致——你不必维护两份内容。

## 为什么不直接给 Markdown？

有人会说：给机器原始 `.md` 不就行了？但 Markdown 里常混着相对链接、未解析的内联
HTML、需要 base path 的图片。先统一渲染成干净 HTML，机器拿到的是**已经解析好、可直接
读取**的结构，省去再解析一遍的麻烦。

> 好的文档，应该让人读得舒服，也让机器读得明白。

这就是 goodoc 想做的小事。
