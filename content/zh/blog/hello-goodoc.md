# 你好，goodoc

我一直想要一个能复用的"项目门面"：一个落地页讲清楚项目是什么，一套文档把用法写明白，再加一处博客记录近况。市面上的方案要么太"科技风"，要么把 Markdown 渲染得很潦草。

于是有了 **goodoc**。

## 它解决什么

- **一个落地页**，用文艺、克制的排版宣传项目；
- **一套文档**，像 GitHub README 一样渲染 Markdown；
- **一处博客**，用同样的方式写随笔与更新。

只有 light theme，字体用 PT Sans 配 PT Serif，整体偏向编辑与纸张的质感，而非冷硬的科技感。

## 一个不一样的地方

goodoc 给每篇文档准备了两个视图：humanize（给人）与 agent（给机器）。这件事我想单独写一篇聊聊——见
[为两种读者而设计](/blog/humanize/designing-for-two-readers/)。

## 现在就开始

```bash
git clone https://github.com/zobinHuang/goodoc.git
```

写下你的第一篇 Markdown，剩下的交给 goodoc。
