# Hello, goodoc

I've long wanted a reusable "project front door": a landing page that explains what the project is, a set of docs that spell out how to use it, plus a blog to record what's new. The options out there were either too "techy" or rendered Markdown sloppily.

So I built **goodoc**.

## What It Solves

- **A landing page**, promoting the project with literary, restrained typography;
- **A set of docs**, rendering Markdown just like a GitHub README;
- **A blog**, written the same way for notes and updates.

It's light theme only, with PT Sans paired with PT Serif, leaning toward the feel of editorial work and paper rather than cold, hard tech.

## One Thing That's Different

goodoc gives every document two views: humanize (for people) and agent (for machines). That's worth a post of its own—see
[Designing for Two Readers](/blog/humanize/designing-for-two-readers/).

## Start Now

```bash
git clone https://github.com/zobinHuang/goodoc.git
```

Write your first Markdown file and leave the rest to goodoc.
